API Reference

This document provides detailed information about the WebSocket API endpoints available in the Smart Reservation System.

WebSocket Connection

All API communication is done through WebSocket connections.

ws://your-server:8000

Enroll Face

WebSocket

Register a new face in the system.

Request

{
    "command": "ENROLL_FACE",
    "frames": 5  // Number of frames to capture
}

Response

{
    "status": "success",
    "message": "All frames received successfully, start enrolling...",
    "face_id": "unique_face_id"
}

Error Response

{
    "status": "error",
    "message": "Not all frames were received, try again"
}

Detect Face

WebSocket

Detect and recognize faces in real-time.

Request

{
    "command": "DETECT_FACE",
    "frames": 1  // Number of frames to process
}

Response

{
    "status": "success",
    "message": "Face detected",
    "face_id": "recognized_face_id",
    "confidence": 0.95
}

Request Status

WebSocket

Get the current status of the system.

Request

{
    "command": "REQUEST_STATUS"
}

Response

{
    "status": "success",
    "message": "Status: OK",
    "system_info": {
        "uptime": "2h 30m",
        "faces_registered": 150,
        "active_connections": 5
    }
}

Shutdown

WebSocket

Gracefully shutdown the server.

Request

{
    "command": "SHUTDOWN"
}

Response

{
    "status": "success",
    "message": "Server shutting down"
}
